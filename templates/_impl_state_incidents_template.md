# インシデント・違反ログ (incidents.md)

> 実装エージェントがルール違反・失敗・自己回復した記録を残すファイル。
> 後で振り返ってルール改善・hook 追加につなげる。

最終更新: <YYYY-MM-DD>

---

## 記録粒度

以下を記録する:

- ルール違反(IMPL_RULES の R-X に違反した、または violation を検知した)
- hook によるブロック(secret_guard/docs_readonly_guard 等)
- エージェントの失敗(例外、リトライ、放棄)
- 並列実行で起きたファイル衝突
- ユーザーへのエスカレーションが必要だったケース

---

## ログ

### INC-YYYY-MM-DD-001 <一行サマリ>

- **発生日時**: YYYY-MM-DD HH:MM
- **エージェント**: <impl-XXX>
- **チケット**: T-XXX
- **違反/失敗種別**: ルール違反 / hook ブロック / 例外 / 並列衝突 / その他
- **詳細**: 何が起きたか
- **影響**: チケット停止 / 別エージェント再アサイン / 仕様確認待ち 等
- **対応**: どう収束させたか
- **再発防止**: ルール強化 / hook 追加 / ドキュメント更新 等の提案
